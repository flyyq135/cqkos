# 重庆口播封面生成

## 目标

把房产口播内容快速转成可交付封面三件套：数据判断型、客户问题型、实地验证/板块对比型。默认追求 `assets/style-reference-*.png` 的商业海报质感：大人物、大标题、金色板块标签、深蓝/橙色强对比、地图/图表/卡片元素。程序化脚本只作为文字可控版和兜底工具，不作为最终审美上限。

## 输入识别

开始前识别这些信息：

- 区域/板块名：如中央公园、礼嘉、悦来。缺失时从标题或口播内容中提取；无法提取时再询问。
- 标题/选题：用于主标题。若用户只给口播内容，从核心判断中提炼 6-10 字标题。
- 口播内容：用于提取客户问题、核心判断、数据标签和适用模板。
- 人物照片：有则使用；无则用剪影占位，并提示后续可替换。
- 账号名称：若用户指定，逐字使用；未指定则默认 `{区域}买房参谋｜XX`。
- 输出目录：未指定时输出到当前项目的 `封面模板/`。

## 工作流

1. 先判断封面目的：板块判断、客户问题、实地验证/板块对比。
2. 读取对应风格参考图：
   - 数据判断型：`assets/style-reference-data.png`
   - 客户问题型：`assets/style-reference-question.png`
   - 实地验证型：`assets/style-reference-field.png`
3. 优先用图片生成工具按参考图风格生成高质感封面：保留大人物、大标题、金色区域标签、地图/图表/卡片、底部账号栏等视觉语言。
4. 从口播内容提取最多 3 个数据/证据标签；没有可靠数字时用“轨道路径、商业半径、挂牌结构、预算、产品、片区”等结构标签。
5. 检查生成图中文字、数字、账号名、合规词、人物遮挡。若 AI 图中文字有错，优先重生成；仍不稳定时，用 `scripts/create_cover_templates.py` 生成可控文字版兜底。
6. 若用户要求同步进 Word/Excel，再用文档/表格工具把图片和规则写入对应模块。

## 图片生成提示词

用图片生成工具时，明确引用对应参考图的视觉标准，并要求：

- 竖版 3:4，1080×1440。
- 画面达到参考图质感，不要像简单 PPT 或程序化草图。
- 区域名和主标题都要大，区域名用金色/橙色标签区分。
- 人物为左侧或右下角真实商务半身照/剪影，西装，专业感。
- 底部账号栏固定为用户给定账号名。
- 中文必须清晰，不要乱码，不要多余文字。
- 不放二维码、电话、微信号、违规营销词。

如果用户给了人物照片，先用该照片作为人物参考；没有人物照片，用参考图中的黑色剪影效果。

## 脚本用法

当图片生成中文字不稳定、用户要求“文字必须完全准确”或需要批量兜底时，运行脚本生成可控文字版：

```bash
python /Users/yq/.codex/skills/cq-koubo-fengmian/scripts/create_cover_templates.py \
  --region 中央公园 \
  --account-line '中央公园买房参谋｜XX' \
  --output-dir ./封面模板
```

可替换标题、副标题和标签：

```bash
python /Users/yq/.codex/skills/cq-koubo-fengmian/scripts/create_cover_templates.py \
  --region 礼嘉 \
  --account-line '礼嘉买房参谋｜XX' \
  --data-title '进入筛选期' \
  --data-subtitle '现在怎么看' \
  --data-tags '去化9个月,参考价15993/㎡,商业兑现85%+' \
  --question-title '220万怎么选' \
  --question-subtitle '先定路径，再看房' \
  --question-tags '自住舒适线,通勤半径线,未来流通线' \
  --field-title '核心区怎么分' \
  --field-subtitle '别只看板块名' \
  --field-tags '轨道路径,商业半径,挂牌结构'
```

如用户提供人物照片，传入：

```bash
--portrait /path/to/person.jpg
```

脚本会把照片裁成居中半身占位效果；如果人物背景复杂或效果不理想，建议先提示“可单独做抠图/裁切/弱化背景”。

如果不是安装在 `/Users/yq/.codex/skills`，把命令中的脚本路径替换为：

```bash
${CODEX_HOME:-$HOME/.codex}/skills/cq-koubo-fengmian/scripts/create_cover_templates.py
```

## 调整规则

- 用户说“区域太大/太小”：只调区域名与主标题比例，不改账号行。
- 用户说“标题不突出”：增强主标题对比度、字重和留白。
- 用户说“太乱”：减少标签、地图线、图形元素和卡片数量。
- 用户说“更专业”：降低饱和度，增加留白、数据卡片和深蓝/金色比例。
- 用户说“更像达人”：增加人物占比和情绪化标题，但不使用违规词。
- 用户说“账号名不要变”：底部账号行完全不动。
- 用户说“文字改成 XX”：逐字替换，不自行改写。

## 合规检查

生成前后都检查：

- 不出现手机号、微信号、二维码、私域引流。
- 不出现必涨、稳赚、抄底、闭眼买、内部房源、学区承诺、制造焦虑、零首付、首付贷、经营贷、包成交、捡漏、大降价。
- 价格、参考价、成交价、挂牌价不要混写；封面如使用数字，必须来自用户材料或已核验资料。

详细版规则见 `references/cover-style-rules.md`。
