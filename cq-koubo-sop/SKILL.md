# 重庆区域口播运营落地包

## 目标

把一个重庆房产区域做成“研究先行 + 日更口播 + 集中拍摄 + 数据复盘”的完整落地包。默认交付 Word 手册和 Excel 执行表，而不是只给方案大纲。

## 输入识别

开始前识别：

- 区域/板块名：如中央公园、礼嘉、悦来、照母山、空港。
- 账号对象：区域总监、大区负责人、商圈经理或区域型顾问；未指定时按“区域总监”。
- 平台：默认视频号优先，抖音/小红书复用。
- 资料来源：官方公开数据、平台参考数据、第三方监测、贝壳内部可用数据、一线客户问题、竞品公开内容。
- 输出目录：未指定时输出到当前项目下 `{区域}口播运营落地包/`。

## 工作流

1. 做研究底座，不先拍脑袋排选题：
   - 数据资料包不少于 40 个数据点。
   - 客户问题池不少于 100 条原始问题。
   - 竞品拆解不少于 50 条样本。
2. 用研究反推内容：
   - 30 天日更排期必须能追溯到数据、客户问题或竞品拆解。
   - 15 条首批脚本必须有证据锚点、核心判断和合规边界。
3. 生成交付文件：
   - Word：账号定位、栏目、封面规范、脚本、拍摄、发布、复盘。
   - Excel：总览、数据资料包、客户问题池、竞品拆解、封面模板、30天排期、15条脚本卡、拍摄通告、发布复盘。
4. 封面模块调用 `cq-koubo-fengmian`：
   - 不在本 skill 重复封面绘图逻辑。
   - 默认输出数据判断型、客户问题型、实地验证/板块对比型三类封面。
5. 做合规检查：
   - 不写收益承诺、成交承诺、教育承诺、违规引流。
   - 价格、参考价、挂牌价、成交价必须区分。

## 生成初稿脚本

当用户需要先生成标准空包或执行框架时，运行：

```bash
python /Users/yq/.codex/skills/cq-koubo-sop/scripts/build_region_koubo_package.py \
  --region 礼嘉 \
  --output-dir ./礼嘉口播运营落地包
```

常用参数：

- `--region`：区域名，必填。
- `--account-line`：账号名，默认 `{区域}买房参谋｜XX`。
- `--owner-role`：账号对象，默认 `区域总监`。
- `--output-dir`：输出目录。
- `--generate-covers`：尝试调用 `cq-koubo-fengmian` 生成封面模板。

如果不是安装在 `/Users/yq/.codex/skills`，把命令中的脚本路径替换为：

```bash
${CODEX_HOME:-$HOME/.codex}/skills/cq-koubo-sop/scripts/build_region_koubo_package.py
```

脚本生成的是结构化初稿。执行正式项目时，要先联网/内部资料补齐研究底稿，再重写排期和脚本，不要把占位内容当成最终研究结论。

## 质量标准

最终交付必须满足：

- Excel 含 9 个工作表，字段完整。
- 数据资料包不少于 40 条，含时间、来源、链接、口径、用途。
- 客户问题池不少于 100 条，能合并出高频母问题。
- 竞品拆解不少于 50 条，只学习结构，不复制原文。
- 30 天选题不少于 30 条，全部有来源类型和证据锚点。
- 首批脚本卡不少于 15 条，每条有完整口播、封面文案、正文、互动话术、合规提示。
- Word 能直接给区域负责人培训使用。
